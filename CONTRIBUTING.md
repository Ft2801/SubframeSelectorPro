# Contributing to SubframeSelectorPro

First off, thank you for considering contributing to SubframeSelectorPro! It's people like you that make SubframeSelectorPro such a great tool for the PixInsight community.

## Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

### Found a Bug?
* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/Ft2801/SubframeSelectorPro/issues).
* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/Ft2801/SubframeSelectorPro/issues/new). Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Missing a Feature?
* You can *request* a new feature by [submitting an issue](https://github.com/Ft2801/SubframeSelectorPro/issues/new) to our GitHub Repository.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Translation Contributions
If you want to add a new language or improve existing translations:
1. Edit `src/SubframeSelectorPro-i18n.js`
2. Add your language to the `languageNames` array
3. Follow the structure of the existing dictionary
